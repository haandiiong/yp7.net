## 使用风险提示

外网应用的可用性受客户端、机场节点、DNS、账号、浏览器、平台状态和本地网络影响。本文只提供常见排查步骤，不保证任何节点或服务长期可用。涉及工作、学习和账号安全时，请保留备用方案。

很多用户会遇到这种情况：Google 能打开但 YouTube 卡，YouTube 能看但 GitHub 拉不动，Gmail 登录异常，或者只有某个软件不走代理。单应用打不开时，不要直接判断“机场坏了”，要先定位是哪一层出问题。

## 快速判断表

| 现象 | 常见原因 | 先做什么 |
|---|---|---|
| Google 打不开 | 代理没开、DNS、规则错误、节点不可用 | 切全局模式，换节点测试 |
| YouTube 卡顿 | 带宽不足、晚高峰拥堵、节点倍率高 | 换地区，避开拥堵节点 |
| GitHub 网页能开但 Git 拉取失败 | 终端未走代理、SSH/HTTPS 配置不同 | 给 Git 或终端单独配置代理 |
| Gmail 登录异常 | Google 账号风控、节点地区变化 | 固定地区，检查账号安全提示 |
| 只有 App 不可用 | App 不读取系统代理或被规则直连 | 开启 TUN 模式或检查分流规则 |
| 开代理后国内网站慢 | 全局代理导致国内流量绕路 | 使用规则模式或绕过大陆地址 |

## 第一步：先测基础连接

按顺序测试：

1. 更新机场订阅。
2. 切换一个常用节点，比如日本、新加坡、美国。
3. 开启系统代理。
4. 先用全局模式打开 Google。
5. 再打开 YouTube、GitHub、Gmail。
6. 如果全局可用，再切回规则模式。

如果全局模式可用、规则模式不可用，问题多半在规则分流或 DNS，而不是机场节点本身。

## Google 打不开

优先检查：

- 系统代理是否开启。
- 浏览器是否使用了独立代理插件。
- Clash 或 v2rayN 是否选中了可用节点。
- DNS 是否被污染或解析失败。
- 是否同时运行多个代理软件。

如果所有节点都打不开 Google，建议先看 [Clash订阅导入失败排查](/posts/clash-subscription-troubleshooting/)。

## YouTube 打不开或很卡

YouTube 更看带宽、丢包和晚高峰容量。处理方式：

- 尝试日本、新加坡、香港、台湾、美国节点。
- 先用 1080p 测试，再试 4K。
- 观察是否只有晚高峰卡。
- 注意机场节点倍率，部分流媒体节点扣流量更多。
- 不要只看 Speedtest，直接播放视频更有参考价值。

需要专门看视频和流媒体，可以参考 [流媒体机场榜](/rankings/streaming/)。

## GitHub 打不开或 Git 拉不动

开发者常见问题是浏览器能打开 GitHub，但终端、IDE、Git、npm、Docker 不走代理。

排查重点：

- 浏览器和终端是否使用同一个代理。
- Git HTTPS 是否配置了代理。
- SSH 连接是否需要单独配置。
- Clash 是否开启 TUN 模式。
- 公司网络、防火墙或 DNS 是否拦截。

如果只是网页浏览，系统代理通常够用；如果是开发工具链，可能需要在工具里单独设置代理。

## Gmail 登录异常

Gmail 与 Google 账号安全策略关联更强。建议：

- 固定常用节点地区，不要频繁跨国家切换。
- 检查 Google 账号安全提示。
- 避免免费共享节点登录重要邮箱。
- 清理异常插件或使用干净浏览器配置。
- 如果 Google 服务大面积异常，先等待平台恢复或换节点对比。

## 常见问题

### 为什么 Google 能开但 GitHub 不能

可能是规则分流、DNS、GitHub 单站点路由、终端代理或节点 IP 问题。先切全局模式测试，再判断是不是规则问题。

### 为什么浏览器能用，App 不能用

很多 App 不一定读取浏览器代理。可以开启 TUN 模式，或检查系统代理和客户端规则。

### 为什么换节点后 Gmail 要验证

Google 可能检测到登录地区或 IP 变化。重要账号建议固定少数稳定节点，避免频繁换地区。

### 一个机场能解决所有应用吗

不一定。Google、YouTube、GitHub、Gmail、ChatGPT、Netflix 的要求不同。依赖工作或学习的用户最好准备备用机场。

## 总结

单个外网应用打不开时，先判断是所有网站都失败，还是只有某个应用失败。全局模式能用但规则模式不能用，优先查规则和 DNS；浏览器能用但 App 不能用，优先查系统代理和 TUN；Google 能用但 AI 工具不能用，再看平台地区和账号风控。

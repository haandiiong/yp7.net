## 测评原则

yp7.net 的机场测评更关注真实使用体验，而不是单次测速截图。机场服务会受用户地区、运营商、晚高峰、线路维护、节点负载和平台策略影响，因此任何结论都需要持续复测。

本站所有推荐都只作为购买前筛选参考，不作为长期稳定、永久可用、绝对安全或固定解锁能力的承诺。第一次使用某个机场时，建议先选择试用、月付或更短周期套餐。

## 测评对象与数据来源

本站会把机场拆成几个维度观察，而不是只看官网宣传：

| 来源 | 用途 | 风险 |
|---|---|---|
| 服务商官网与后台 | 核对套餐、价格、流量、试用、客户端和退款规则 | 规则可能临时调整 |
| 实际使用体验 | 观察网页、ChatGPT、视频、远程办公和长时间连接 | 受地区、运营商和设备影响 |
| Speedtest 与客户端延迟 | 做速度和延迟初筛 | 单次结果容易失真 |
| YouTube 4K 与流媒体播放 | 判断真实视频体验和解锁能力 | Netflix、Disney+、TikTok 策略会变化 |
| 用户反馈与风险记录 | 发现官网异常、客服失联、节点波动和套餐变化 | 个别反馈需要复核 |
| 站内结构化数据 | 同步榜单、风险监测、价格和试用信息 | 需要持续维护 |

如果公开信息、后台规则和实际测试结果冲突，本站会优先提示不确定性，并建议读者在自己的设备和常用时间段复测。

## 核心指标

| 指标 | 观察内容 | 说明 |
|---|---|---|
| 晚高峰稳定性 | 20:00-23:00 网页、视频、ChatGPT 是否稳定 | 比白天测速更重要 |
| 延迟 | Clash 或客户端延迟测试 | 只做初筛，低延迟不等于高速 |
| 下载速度 | Speedtest、实际下载、视频缓冲 | 观察真实吞吐 |
| 流媒体 | YouTube 4K、Netflix、Disney+、TikTok | 解锁能力会变化 |
| ChatGPT | 登录、连续对话、断连情况 | 适合办公用户重点测试 |
| 客户端 | 专属客户端、Clash、Shadowrocket、Mihomo、sing-box | 决定上手成本和灵活性 |
| 价格与流量 | 最低月付、月流量、试用、不限时套餐 | 不只看最低价 |
| 风险 | 官网、售后、退款、支付、历史异常 | 决定是否适合长期付费 |

## 评分权重

机场推荐页和榜单页不会只按最低价格或单次测速排序。综合推荐时，本站默认参考以下权重：

| 维度 | 权重 | 为什么重要 |
|---|---:|---|
| 晚高峰稳定性 | 30% | 20:00-23:00 更容易暴露拥堵、掉线、节点不可用和线路绕路 |
| 新手易用性 | 20% | 专属客户端、订阅导入、节点切换和教程完整度会影响上手成本 |
| ChatGPT / 办公体验 | 20% | 连续对话、网页加载、远程办公和常用工具访问更接近真实需求 |
| 视频与流媒体体验 | 15% | YouTube 4K、Netflix、Disney+、TikTok 等场景更容易暴露节点质量 |
| 价格与购买风险 | 15% | 价格、试用、退款、客服、支付和历史异常决定是否适合长期续费 |

不同榜单会调整重点。例如 [销量机场榜](/rankings/sales/) 会更看重本站当前可见销量样本，[低价机场榜](/rankings/cheap/) 会更看重价格和试错成本，[ChatGPT机场榜](/rankings/chatgpt/) 会更看重 AI 工具访问稳定性，[流媒体机场榜](/rankings/streaming/) 会更看重地区节点和播放体验。

## 推荐测试流程

1. 先看 [全量机场榜单](/rankings/all/) 缩小候选范围。
2. 根据场景查看 [销量机场榜](/rankings/sales/)、[稳定机场榜](/rankings/stable/)、[低价机场榜](/rankings/cheap/)、[免费试用机场榜](/rankings/trial/)、[不限时机场榜](/rankings/no-expiry/)、[专属客户端机场榜](/rankings/dedicated-client/)、[Clash机场榜](/rankings/clash/)、[ChatGPT机场榜](/rankings/chatgpt/) 或 [流媒体机场榜](/rankings/streaming/)。
3. 阅读单个机场测评页，确认价格、试用、客户端和风险提示。
4. 购买月付、试用或短周期套餐。
5. 在自己的常用设备、地区、运营商和晚高峰时间段测试。
6. 确认稳定后再考虑更长周期套餐。

## 证据记录方式

本站更新机场评测时，会尽量记录这些信息：

- 测试日期和页面更新时间。
- 测试时段，尤其是 20:00-23:00 晚高峰。
- 客户端类型，例如专属客户端、Clash Verge、Clash Meta、Shadowrocket、Mihomo 或 sing-box。
- 套餐价格、月流量、试用规则、不限时规则和优惠码。
- ChatGPT、YouTube 4K、Netflix、Disney+、TikTok 等典型场景。
- 官网入口、客服响应、退款说明、支付方式和历史异常。

测速截图、延迟截图和流媒体截图只作为辅助证据。由于同一机场在不同地区、运营商、设备和时段下差异很大，本站不会把单张截图当作最终结论。

## 商业披露和排序规则

部分服务商入口可能包含邀请码、推广参数或联盟追踪参数。这类链接不会改变排序原则，构建时会自动标记为 `rel="sponsored nofollow noopener noreferrer"`。

本站不建议因为返利、优惠码、低价或短期活动直接年付。推荐顺序主要依据晚高峰稳定性、真实使用体验、客户端兼容性、适合人群和购买风险，而不是推广佣金。

更完整的站点说明见 [关于 yp7.net](/about/)。

## 不采用的判断方式

- 不只按最低价格排序。
- 不只按单次 Speedtest 结果排序。
- 不把服务商宣传文案当作稳定性证据。
- 不承诺任何机场长期永久可用。
- 不把“节点数量多”直接等同于“可用节点多”。
- 不把“支持流媒体”直接等同于“长期稳定解锁”。
- 不把“有优惠码”直接等同于“更值得购买”。

## 页面更新优先级

本站会优先更新这些变化：

| 变化类型 | 更新方式 |
|---|---|
| 官网入口失效 | 优先在风险提示和风险监测页标记 |
| 套餐价格变化 | 同步单页测评、机场大全和相关榜单 |
| 试用或优惠码变化 | 同步免费试用榜、优惠码页和对应测评 |
| 客服失联或退款异常 | 加入风险监测，并在相关页面提醒 |
| 流媒体或 ChatGPT 大面积异常 | 更新场景榜单和主推荐页 |
| 新增机场或淘汰机场 | 同步结构化数据、榜单和 sitemap |

## 常见问题

### 为什么晚高峰测试最重要？

晚高峰用户最多，更容易暴露节点拥堵、线路绕路、售后维护和实际稳定性问题。

### 为什么测评结论会变化？

机场节点、套餐、优惠码、官网入口、支付方式和流媒体策略都会变化，因此测评需要持续更新。

### 为什么不建议直接年付？

机场服务受线路成本、节点维护、用户数量、平台风控和服务商运营影响。第一次使用时直接年付，会把不确定性放大。更稳妥的做法是先试用或月付，确认自己的常用场景稳定后再决定。

### 为什么不同用户体验差异很大？

同一个节点在电信、联通、移动、校园网、公司网络和不同地区的表现都可能不同。设备系统、客户端版本、DNS、规则分流和本地网络质量也会影响结果。

### 为什么榜单不只按速度排序？

速度只是体验的一部分。长期稳定性、客户端易用性、售后、退款、试用、支付风险和流媒体策略变化，同样会影响是否值得购买。

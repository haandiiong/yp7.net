Clash Verge 是一款基于 Clash 内核的跨平台图形化客户端，专注于订阅管理、节点切换与规则分流，支持 Windows、macOS 与 Linux。本文为你提供快速上手指南、常见场景示例以及 SEO 优化的实用关键词（Clash Verge 安装、Clash Verge 配置、订阅导入、代理分流）。

<!-- more -->

## 为什么选择 Clash Verge？（关键优势）

- 跨平台：一次学习，多平台使用（Windows / macOS / Linux）。
- 图形化操作：无需频繁手动编辑 YAML，适合不熟悉命令行的用户。
- 订阅管理：支持一键导入机场订阅并定期刷新节点列表。
- 策略分流：支持按域名、IP、应用或自定义规则进行流量路由。

这些特性使 Clash Verge 成为管理多个机场订阅和在多系统间统一配置的实用工具。

## 主要功能速览

- 订阅导入与自动更新
- 节点选择、测速与优先级设置
- 策略组（Policy）与规则（Rule）编辑
- 支持多种代理协议（Vmess/VLESS/SOCKS 等）
- 支持自定义 DNS 与本地代理端口设置

## 安装（快速指南）

### Windows / macOS / Linux

1. 前往项目发布页或官网下载对应平台的安装包/压缩包。
2. 解压并运行应用（Windows 可直接运行 exe，macOS 需允许运行未签名应用，Linux 可用 AppImage 或二进制）。
3. 第一次运行时允许网络权限，按向导完成基本配置。

注意：不同平台的 UI 细节略有差异，但订阅导入与规则管理逻辑一致。

## 快速配置：导入订阅与启用节点

1. 在机场控制台复制订阅链接（通常为 `http(s)://.../subscribe`）。
2. 打开 Clash Verge，找到“订阅/Profiles”页面，粘贴订阅 URL 并保存。
3. 刷新订阅，选择一个延迟低、状况正常的节点并启用。
4. 在“模式”中选择合适的代理模式（全局 / 规则 / 直连）。

优化小贴士：导入后可使用内置测速或外部 speedtest 对比不同节点延迟与下载速度，优先选择延迟低且稳定的节点。

## 策略与规则建议（分流设置）

- 推荐策略：将常访问的国内站点设置为直连，将需要翻墙的流量走规则代理。
- 使用策略组（如 `Auto`, `Proxy`, `Direct`）配合规则实现自动分流。
- 对常用服务（Netflix、Google、YouTube）创建针对性规则以提高命中率。

示例规则顺序（建议）：

1. GEOIP / 国内 IP 列表 -> DIRECT
2. 特定域名（国内） -> DIRECT
3. 常用翻墙域名 -> Proxy
4. 其他 -> Auto/Proxy

## 常见问题（FAQ）

- **Q：如何在 Clash Verge 中切换代理模式？**
  - A：主界面或系统托盘图标处可直接选择“全局/规则/直连”，也可在设置中修改默认模式。

- **Q：订阅导入后节点无法连接怎么办？**
  - A：先尝试刷新订阅并测试其他节点；如仍不行，检查订阅源是否过期或与机场沟通获取最新订阅地址。

- **Q：如何让特定程序走代理？**
  - A：通过策略组与进程名或端口规则（部分客户端支持按进程分流）实现；若不支持可使用系统代理或第三方工具配合。
